## Design Context

### Users
Music producers, sound designers, and electronic musicians using Freeze95 inside a DAW. They want fast access to a powerful DSP without needing to decode a dense control surface.

### Brand Personality
Old-computer hardware, glitchy, professional. The interface should feel like compromised late-90s or early-2000s studio hardware that still behaves with modern precision.

### Aesthetic Direction
Preserve the retro monitor and hardware shell language. Follow the Cloph visual manifesto: vector-grounded skeuomorphism, macro-tactility, asymmetric experimentalism, and qualitative telemetry. Freeze95 should feel like a self-contained micro-world rather than a generic plugin panel.

### Design Principles
- Keep the UI to a small set of macro controls with immediate, obvious roles.
- Use spacing to separate functional groups: texture shaping, clock control, and power state.
- Preserve retro character, but remove any crowding that slows down workflow.
- Favor asymmetric compositions and intentional dead space over equalized, generic alignment.

Old computer hardware, glitchy, professional. The interface should feel like a purposeful retro instrument rather than a generic plugin panel.

Preserve the retro monitor and hardware look, with vector-based skeuomorphic surfaces, asymmetric composition, and a compact macro-control workflow. The visual language should feel like early-2000s experimental software filtered through rigorous modern execution.

### Design Principles
- Keep the interface simple and macro-focused: three or four primary controls at most, with supporting telemetry kept secondary.
- Use spacing to create clear bands of hierarchy rather than equal density everywhere.
- Preserve the retro hardware narrative with tactile plates, beveled surfaces, and qualitative state feedback.
- Favor asymmetry and purposeful offsets over perfectly centered or grid-like arrangements.
- Keep the UI efficient and readable so the layout supports fast sonic decision-making instead of decoration.
