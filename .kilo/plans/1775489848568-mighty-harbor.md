# Plan: Improve High-Frequency Resolution in RussetNoise

## Problem Analysis

User asked: "Since we are using FFT with 16kHz max, are we losing any information?"

**Answer:** There is NO 16kHz hard limit in the FFT. The FFT goes to Nyquist (22.05kHz at 44.1kHz SR, 24kHz at 48kHz SR).

The actual limitation is **frequency resolution**, which is determined by FFT size:
- FFT 2048 (default): ~21.5 Hz/bin - only ~4-5 bins cover 10kHz-22kHz
- FFT 4096: ~10.8 Hz/bin  
- FFT 16384: ~2.7 Hz/bin - full resolution

## Proposed Options

### Option A: Increase Default FFT Size
- Change default from 2048 to 4096 or 8192
- Tradeoff: Higher latency and CPU usage
- Benefits: Better high-frequency resolution, smoother EQ curves

### Option B: Keep FFT Size but Use Smarter Band Allocation
- Increase band density in high frequencies (more than 64 bands)
- Already using logarithmic spacing, could bias more toward HF
- Benefits: No latency increase
- Drawback: Still limited by underlying bin resolution

### Option C: Hybrid Approach
- Use larger FFT for analysis (4096+) 
- Keep hop size manageable
- Document the resolution vs latency tradeoff

### Option D: Add Sample Rate Warning/Handling
- Detect sample rates below 44.1kHz and warn user
- Inform that HF resolution is limited below Nyquist

## Recommendation

Option A with fallback to Option D:
1. Increase default FFT size to 4096 (good balance of resolution/LATENCY)
2. Add informational UI text showing current HF resolution
3. For sample rates < 44.1kHz, show warning

## Implementation Tasks
