# Fix GUI Lag with Large FFT - Improved Solution

## Problem
The current solution (skip N hops) makes visualizer updates too infrequent for large FFTs, making the plugin appear frozen or crashed.

## New Approach: Smooth Interpolation with Fixed Update Rate

### Key Insight
Instead of skipping based on FFT size, we:
1. Update visualizer data at a FIXED target rate (e.g., 30 FPS = every ~33ms)
2. Interpolate/smooth the display between updates to look smooth even with sparse data

### Technical Solution

**Option A: Time-Based Update (Recommended)**
- Track time since last visualizer update
- Update at fixed interval (~30ms = 33 FPS) regardless of FFT size
- Interpolate between last two data points in the Draw() method for smooth animation

**Option B: Latching Data + Smooth Draw**
- Keep latest spectrum data in a thread-safe way
- In Draw(), apply temporal smoothing/lerping to make old data gradually move toward new data
- Even 2-3 updates per second will look smooth because of the interpolation

### Implementation

**Step 1: Add timing/state to visualizer**
- Add timestamp of last update
- Add interpolation alpha based on time since last update

**Step 2: Modify Draw() to interpolate**
- Store previous frame's data
- Lerp between previous and current based on elapsed time
- This makes sparse updates appear smooth

**Step 3: Target 30 FPS updates**
- Instead of `skipCount = mFFTSize / 2048`
- Use: `skipCount = max(1, (44100 / hopSize) / 30)` to target 30 FPS

This gives:
| FFT | Hops/sec | Skip for 30 FPS |
|-----|----------|-----------------|
| 256 | 344 | 11 |
| 512 | 172 | 5 |
| 1024 | 86 | 3 |
| 2048 | 43 | 1 |
| 4096 | 21 | 1 |
| 8192 | 11 | 1 |
| 16384 | 5.4 | 1 |

### Additional Optimization
Make the visualizer Draw() itself faster:
- Simplify gradient fills (use solid colors instead)
- Reduce number of lines drawn
- Only redraw when actually needed (check dirty flag)

## Expected Result
- GUI stays responsive at 30+ FPS
- Visualizer animation looks smooth due to interpolation
- Audio quality unchanged (full FFT processing)
- No appearance of "crashing" or "frozen" UI
