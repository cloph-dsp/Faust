# Plan: Add Q Knob to RussetNoise

## Goal
Add a Q (bandwidth) knob to control the correction smoothing radius in the RussetNoise plugin, giving users finer control over how narrowly or broadly the spectral correction is applied.

## Current State
The plugin uses `SmoothBandCorrections` with a radius computed from the Smoothing parameter:
```cpp
const int smoothingRadius = static_cast<int>(std::lround(std::pow(smoothing, 0.85f) * 6.f));
```
This gives a range of 0-6 bands of smoothing.

## Implementation

### 1. Add parameter enum
In `RussetNoise.h`, add to `EParams`:
```cpp
kQ = 4,
kNumParams = 5
```

### 2. Add member variable
In `RussetNoise` class private section:
```cpp
float mCurrentQ = 1.f;
```

### 3. Initialize parameter
In `RussetNoise.cpp` constructor, add:
```cpp
GetParam(kQ)->InitDouble("Q", 0.5, 0.01, 4.0, 0.01);
```

### 4. Use Q in ProcessHop
Replace the computed radius with Q-based calculation:
```cpp
// Q controls the smoothing radius (bandwidth of correction)
// Low Q (0.01) = very precise, narrow corrections (CPU intensive)
// High Q (4.0) = broad, heavily smoothed corrections (less CPU)
const float qValue = static_cast<float>(GetParam(kQ)->Value());
const int smoothingRadius = static_cast<int>(std::lround(qValue * 6.f));
// Range: Q=0.01 → ~0 bands (precise), Q=4.0 → 24 bands (broad)
```
**Note**: Higher Q = more smoothing = fewer unique correction values to compute = less CPU. Lower Q = precise per-band corrections = more CPU.

### 5. Add UI control
In the `mLayoutFunc` lambda, add a new IVKnobControl for the Q parameter alongside Amount, Target, and Smoothing.

## Testing
- Build and verify no compilation errors
- Verify Q knob appears in UI
- Test at low Q (precise, narrow corrections) vs high Q (broad, smoothed corrections)
- Verify CPU usage differs: low Q = more computation, high Q = less computation
